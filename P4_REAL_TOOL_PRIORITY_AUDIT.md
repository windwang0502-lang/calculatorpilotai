## Scoring Methodology

Each page scored 1-100 based on weighted criteria:

| Criterion | Weight | Description |
|-----------|--------|-------------|
| **Search Intent Strength** | 25% | How strong is the user's intent to use a calculator? |
| **Business/AdSense Value** | 20% | Revenue potential from ads and user engagement |
| **User Utility** | 20% | How useful is this tool to real users? |
| **Implementation Difficulty** | 15% | Lower difficulty = higher score (inverse scoring) |
| **Low YMYL Risk** | 10% | Lower medical/financial risk = higher score |
| **Category Growth Value** | 10% | Strategic value for category expansion |

### Scoring Scale

- **90-100:** Exceptional - Build immediately
- **80-89:** Excellent - High priority
- **70-79:** Good - Medium priority  
- **60-69:** Fair - Lower priority
- **Below 60:** Defer - Build much later

## TOP 5 RECOMMENDED TOOLS TO BUILD FIRST

### 1. Compound Interest Calculator (Finance)

**Score: 95/100**

**Why Build First:**
- Highest utility score across all categories
- Educational goldmine - "8th wonder of the world" appeal
- Medium competition = achievable ranking
- Clear, deterministic calculation logic
- Excellent link-bait for finance category
- Very low YMYL risk (pure math, no financial advice)

**Expected Inputs:**
- Principal amount ($)
- Interest rate (%)
- Time period (years/months)
- Compounding frequency (daily/weekly/monthly/quarterly/annual)
- Optional: monthly contributions

**Expected Outputs:**
- Future value
- Total interest earned
- Year-by-year growth chart
- Comparison with simple interest

**Calculation Logic:**
```
FV = P(1 + r/n)^(nt) + PMT × [((1 + r/n)^(nt) - 1) / (r/n)]
```
Where: P=principal, r=rate, n=compounds/year, t=years, PMT=monthly contribution

**Related Pages to Link From:**
- savings-accounts, investment-basics, retirement-planning
- finance-pages: loan-types-explained, compound-interest-guide
- health-pages: protein-calculator (linking FROM compound interest)

**Implementation Difficulty:** EASY
**Estimated Build Time:** 4-6 hours

### 2. Calorie Calculator (Health)

**Score: 92/100**

**Why Build First:**
- Universal appeal - everyone thinks about food/weight
- High SEO competition but achievable with quality
- Clear inputs/outputs (Mifflin-St Jeor formula)
- Supports related tools (BMI, macros)
- Strong internal linking potential

**Expected Inputs:**
- Age
- Gender (male/female)
- Height (cm/ft-in)
- Weight (kg/lbs)
- Activity level (1.2 - 1.9 multiplier)

**Expected Outputs:**
- BMR (Basal Metabolic Rate)
- TDEE (Total Daily Energy Expenditure)
- Calories for weight loss (-500 kcal)
- Calories for weight gain (+500 kcal)
- Macro breakdown suggestion

**Calculation Logic:**
```
Male: BMR = 10×weight + 6.25×height - 5×age + 5
Female: BMR = 10×weight + 6.25×height - 5×age - 161
TDEE = BMR × Activity Multiplier
```

**Related Pages to Link From:**
- diet-plans, fitness-routines, weight-loss-tips
- health-pages: macro-nutrients-guide, understanding-bmi
- finance-pages: health-insurance-costs (linking FROM calorie calc)

**Implementation Difficulty:** EASY
**Estimated Build Time:** 5-7 hours

**YMYL Note:** Add disclaimer "This calculator provides estimates only. Consult a healthcare professional for personalized advice."

### 4. Work Hours Calculator (Time)

**Score: 91/100**

**Why Build First:**
- Universal employment relevance
- Clear payroll/time-tracking use case
- High practical utility for workers
- Low competition despite high utility
- Excellent for AdSense (workers/employers)
- Simple calculation logic

**Expected Inputs:**
- Clock in time (or date + time)
- Clock out time (or date + time)
- Break duration (minutes)
- Hourly rate (optional)
- Multi-day support (week view)

**Expected Outputs:**
- Total hours worked
- Regular hours vs overtime
- Earnings estimate (if rate provided)
- Weekly/monthly totals
- Break deduction breakdown

**Calculation Logic:**
```
Work hours = (clock_out - clock_in) - break_duration
Overtime = max(0, work_hours - 40) [or 8 hours/day]
Earnings = (regular_hours × rate) + (overtime × rate × 1.5)
```

- Supports shipping category expansion

**Expected Inputs:**

**Expected Outputs:**

```
Dimensional Weight = (L × W × H) / DIM factor
Billable Weight = max(actual, dimensional)
Rate = base_rate + (weight × price_per_lb) + fuel_surcharge
```
